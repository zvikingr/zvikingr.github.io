前言

什么是音视频通信，有哪些方式，需要什么东西支撑

5G 网络里怎样实现两个终端音视频通信：

  • 使用 APP（微信、钉钉、portsip 等），这些主要是数据面的交互，核心网不需要配合改动。
  • 使用系统拨号盘，涉及到 IMS，需要核心网配合提供接口或数据交换，信令面和基站都需要配套支持。

区别：

微信音视频靠互联网数据传输，可以是 WiFi、宽带、运营商移动网络，需要下载微信客户端。 IMS 不需要手机安装额外客户端，运营商做了很多绑定，比如手机号账号信息、套餐，提供更多收费服务。 另外 IMS 借助运营商网络，支持更多的 Qos 控制，可提供更好的网络质量。

总体架构

宏观图，有个印象，不要细节

什么是 IMS，IP Multimedia Subsystem（IP 多媒体子系统）。

整个 IMS 系统分成拜访网络 (Visited Network) 和归属网络 (Home Network) 两部分。

下图是一个运营商级别的复杂的架构图，存在漫游的情况，方便理解为什么需要三个 CSCF 以及实际用途。

HSS：Home Subscriber Server 归属用户信息服务。

CSCF：全称为 Call Session Control Function（呼叫会话管理系统），负责呼叫的建立、修改、终止以及会话的控制和管理。CSCF 存在三种类型：P-CSCF（Proxy CSCF）、I-CSCF（Interrogating CSCF）和 S-CSCF（Serving CSCF）。

P-CSCF 属于拜访网络，I-CSCF 和 S-CSCF 属于归属网络。

P-CSCF 和 I-CSCF 可认为是两层 NGINX 代理，做的事情很少，核心业务在 S-CSCF，实例数量大概是 1:1:N。

P-CSCF 负责将注册信息转发给 I-CSCF，I-CSCF 负责将请求路由到某一个 S-CSCF。

S-CSCF 也可以将请求路由到下一个网络的 P-CSCF。

通信过程

前提条件：

  • 终端和基站需要支持并开启 VONR
  • 需要 ims 和核心网互通（用于认证校验数据）
  • 需要配置 HSS 建立 IMSI 和手机号的关联关系

上线注册

整体流程：

PDU 会话（ims / internet）随机创建，不分先后

ims PDU 会话：用于终端和 ims 服务通信使用，传递 ims 信令和音视频数据。

internet PDU 会话：用于终端走数据流量使用，如手机上网，刷抖音。

ims PDU 会话建立完成后，终端接着发起了 ims 注册请求。

ims PDU 会话建立成功后，核心网返回给终端以下关键信息：

终端的 IMS IP 地址

IMS 的入口地址 P-CSCF IP

终端允许的上下行带宽

终端拿到 IMS 的注册地址（P-CSCF 的 IP）后，发起 IMS 注册请求（SIP REGISTER 请求）

请求依次经过 UE --> UPF --> P-CSCF --> I-CSCF --> S-CSCF

192.168.190.1:41061 是 P-CSCF 请求 I-CSCF 时的客户端地址

sngrep 抓包看到的 ims 注册流程：

第一个注册包关注的 sip 头域：

From、To：注册时是终端自己的标识

Security-Client：表明客户端支持的加密算法，将要使用的通信端口

Expires：客户端期望的注册过期时间，没什么用，最终由服务端说了算

IMS 返回给终端 401 挑战消息。主要关注的 sip 头域：

Security-Server：表明 IMS 服务端将要使用的通信端口

WWW-Authenticate：给客户端计算认证信息使用的参数

第二个注册包主要关注的 sip 头域：

Security-Client：表明客户端支持的加密算法，将要使用的通信端口

Authorization：包含经过服务端指定的验证算法和参数，以及终端自己的认证信息计算后得到的认证信息

注册成功消息里主要关注的 sip 头域：

Contact 里的 expires：告诉终端本次注册的过期时间。终端会在过期之前发起重注册，一般是取 expires 值的二分之一后发起重注册。

SIP 401 主要是为了让终端使用服务端指定的加密方式进行认证。那么为什么不在第一次注册包里就带上认证消息，原因如下：

第一次认证时终端和 IMS 服务端还没商量好使用哪种认证算法。

防止重放攻击，第二次认证的时候，终端会使用 IMS 服务端返回的 WWW-Authenticate 里面的认证参数来进行认证，特别是其中的 nonce 值，这个值只能使用一次，每次认证都不一样。

发起通话

一个 PDU 会话里面包含多条 QoS 规则，每个规则认为是一个承载。

一个 PDU 会话会有一个默认承载，像音视频场景，会有针对语音和视频的专有承载。

注册成功后，S-CSCF 会定期向 UE 发送心跳消息（Keep Alive 或 Expires Timer 刷新），以维持注册状态。同时，当 UE 的网络位置发生变化时，会重新触发 REGISTER 消息的发送，更新其在网络中的位置信息。

上线失败 401

核心网和 ims 中的 sqn 序列号不同步，导致 IMS 无法上线，一直触发 401 状态码，需要从核心网数据库中把对应终端的序列号写到 HSS。
